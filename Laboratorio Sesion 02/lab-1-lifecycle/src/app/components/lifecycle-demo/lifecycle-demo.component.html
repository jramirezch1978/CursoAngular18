🎯 Lifecycle Hooks Implementados

🔧

constructor()

Propósito: Inyección de dependencias únicamente

Timing: Antes de cualquier inicialización

Uso: Asignar servicios, variables básicas

🔄

ngOnChanges()

Propósito: Reaccionar a cambios en @Input

Timing: Cada vez que cambian las propiedades input

Uso: Validar datos, transformar inputs

🚀

ngOnInit()

Propósito: Inicialización principal

Timing: Una vez después de primer ngOnChanges

Uso: HTTP calls, subscripciones, setup

📝

ngAfterContentInit()

Propósito: Contenido proyectado listo

Timing: Después de proyectar contenido

Uso: Interactuar con ng-content

👁️

ngAfterViewInit()

Propósito: Vista completamente inicializada

Timing: Después de renderizar vista

Uso: Manipular DOM, @ViewChild

🧹

ngOnDestroy()

Propósito: Limpieza de recursos

Timing: Antes de destruir componente

Uso: Unsubscribe, clearInterval

⚠️ Mejores Prácticas y Errores Comunes

✅ SÍ HACER

  • Usar constructor solo para inyección de dependencias
  • Hacer HTTP calls en ngOnInit
  • Limpiar subscripciones en ngOnDestroy
  • Usar ngOnChanges para validar @Input

❌ NO HACER

  • HTTP calls en constructor
  • Olvidar implementar ngOnDestroy
  • Manipular DOM antes de ngAfterViewInit
  • Subscripciones sin unsubscribe
